<https://nemo.seqpipe.org/job/iossifovlab/job/gain/job/master/34/>

None of these are pytest failures. They're lint / type-check

  `Q000=215`, `COM812=141`, `I001=49`, `RUF012=48`, `ARG001=32`,
  `INP001=21`, `E501=19`, `E302=17`, `ARG002=17`, `RET504=9`,

Root cause is consistent for ruff: Phase 2 reconciled the
project-wide `ruff.toml` to enable a wide rule set
(`E,F,W,C90,I,UP,ASYNC,S,BLE,FBT,B,A,COM,C4,T10,EXE,ISC,LOG,G,

FURB,RUF`). The previous `web_infra`-driven pipeline never
applied this rule set to `web_api`. Phase 4 just plugged
`web_api` into the unified pipeline; the lint debt was always
there, now it's visible.

Phase 4.5 makes master green by paying down that debt
(consistent with Phase 2's one-rule-set intent), so Phase 5

## Approach

### Ruff cleanup

From the repo root:

```bash
ruff check --fix --unsafe-fixes web_api/
ruff check web_api/   # see what's left
```

The remaining ~50–80 findings need manual attention:

- **`INP001`** (implicit-namespace-package, ~21 hits): these
  are Django app directories, management/commands subtrees,
  and similar that genuinely have no `__init__.py`. Add empty
  `__init__.py` files where Django doesn't object; for the
  ones Django actively manages by other means, add a per-file
  `# ruff: noqa: INP001` at the top with a one-line reason or
  list them under `[per-file-ignores]` in the root
  `ruff.toml` keyed on the specific paths.
- **`ARG001`/`ARG002`** (~49 hits): most are unused
  `request`/`*args`/`**kwargs` in Django view methods and
  signal handlers. Rename to `_request` / `_args` /
  `_kwargs`, or add a `# noqa: ARG002` for cases where the
  signature is fixed by Django/DRF. Prefer rename; the
  signature-locked cases are the minority.
- **`RUF012`** (~48 hits): Django model `Meta`, DRF
  `Serializer.Meta`, view `permission_classes`,
  `authentication_classes`, etc. — all should be annotated as
  `ClassVar[...]` (Django's runtime is fine with that).
  `ruff --fix` won't do this automatically because it
  requires reading the surrounding type. Apply by hand.
- **`S106`** (~6 hits): hardcoded password strings — virtually
  always inside test files. Add `# noqa: S106` per line with
  a comment, or list the test file under `[per-file-ignores]`
  for `tests/**/*.py`.
- **`PT011`** (~9 hits): `pytest.raises(ValueError)` without a
  `match=` argument. Add `match=` patterns where the original
  code had a specific message in mind; otherwise narrow the
  exception type or accept and `# noqa`.
- **`ERA001`** (~3 hits): commented-out code. Delete the
  commented blocks unless they're load-bearing TODO markers
  (rare).
- **`BLE001`** (~4 hits): `except Exception:` blanket catches.
  Replace with the specific exception type if obvious;
  otherwise `# noqa: BLE001` with a one-line reason.
- Other singletons (`TRY300`, `SLF001`, `SIM300`, `FBT001/2`):
  fix by hand.

If the long tail genuinely doesn't fit `web_api`'s style (e.g.,
`FBT001/002` boolean-positional rules that Django REST
framework signatures depend on), add a tightly-scoped
`[per-file-ignores]` entry in the root `ruff.toml` rather than
spamming `# noqa`. Use this sparingly: per Phase 2's intent,
the rule set is the rule set.

   Add `types-requests` to `web_api/pyproject.toml` under the

3. **`export_quotas.py:59-61`** — the loop iterates over a

   into two passes (one per concrete type) or add

4. **`single_allele_annotation/views.py:223,228`** —
   `request.user.as_owner` returns `User | str`, but the
   `AlleleQuery.owner` lookup expects `User | int | None` /

   `as_owner` model property to return `User | int | None`
   (likely the right fix — `str` was probably a session-key
   placeholder that should be the anonymous user id, or
   `None`), or (b) coerce at the call site. Prefer (a) once
   the intended semantics are confirmed by reading
   `models.py:67` and the `as_owner` definition.

### pylint fix

Hypothesis (verify before committing): pylint imports
`web_annotation` to introspect; without `pylint-django`
configured against a settings module, Django models fail to
import (or AppRegistryNotReady fires) and pylint dies before
the `pylint_junit` reporter can write a header — yielding a
0-byte file.

The fix is three pieces:

1. **Add `pylint-django` to web_api dev-deps** in
   `web_api/pyproject.toml`.
2. **Load it in `web_api/pylintrc`** (file already exists for
   `extension-pkg-allow-list`):
   ```ini
   [MASTER]
   load-plugins=pylint_django
   django-settings-module=web_annotation.test_settings
   ```
3. **Have `runProject()` honour per-project `pylintrc`**:
   today line ~42 of root `Jenkinsfile` does
   `pylint --rcfile=/workspace/pylintrc ...`. Change it to
   prefer `/workspace/<name>/pylintrc` when it exists,
   falling back to the root one. This mirrors how
   `mypyExtra: '--config-file /workspace/web_api/mypy.ini'`
   threads a per-project mypy config today, but does it
   automatically based on file presence rather than via a
   per-call argument.

If `pylint-django` still can't import the settings module
without an env var (Django apps with database models often
need `DJANGO_SETTINGS_MODULE` set in the *process* env, not
just in `pylintrc`), add `-e DJANGO_SETTINGS_MODULE=
web_annotation.test_settings` to the `web_api` stage's
`dockerRunExtra` in the root `Jenkinsfile`. (The
`runProject()` call already has a `dockerRunExtra` slot used
for `-e GPFWA_EMAIL_HOST=mail`.)

Sanity check after the fix: `reports/web_api/pylint.xml`
should be a non-zero, well-formed JUnit XML —
`<testsuites><testsuite/></testsuites>` at minimum if pylint
reports nothing — and the JUnit publisher should stop
flagging it as length-0.

## Risks and known unknowns

- **Auto-fix scope creep.** `ruff --fix --unsafe-fixes`
  occasionally rewrites `RUF010` / `UP035` in surprising
  ways. Review the diff before committing rather than
  trusting it blindly.
- **`pylint-django` requires Django apps to load.** If the
  settings module pulls in resources that aren't installed
  in the CI image (e.g., a real database), pylint may still
  fail. Falling back to a minimal pylint-only settings module
  may be necessary; Django ships `manage.py test` settings
  that work for this. The `web_annotation.test_settings`
  module is the first candidate; a thinner `pylint_settings`
  is the fallback.
- **`INP001` on Django dirs.** Django app directories
  intentionally lack `__init__.py` in some cases (for
  namespace-package patterns). If adding `__init__.py`
  breaks app discovery, fall back to `[per-file-ignores]`
  for that specific subtree.
- **`as_owner` model fix.** Changing the return type of
  `as_owner` from `User | str` to something that satisfies
  the lookup may ripple into call sites that depend on the
  string form (session keys for anonymous users). Trace
  callers before changing the signature; if rippling is too
  wide, coerce at the call site instead.
- **No new ruff/mypy errors introduced.** After the sweep,
  re-run the full `gain-web-api-ci` lint commands and
  diff `reports/web_api/{ruff,mypy}.xml` against expectation
  (zero failures). The Jenkins JUnit publisher will catch
  regressions, but local verification is faster than the
  master loop.
